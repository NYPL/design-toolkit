General tables

Go to top

Use tables sparingly, only for tabular data. Use the nypl-basic-table class to format a table. All tables should have a thead element with the corresponding headers.

Example
One Two Three Four
some value some value some value some value
some value some value some value some value
… … … … 

.nypl-basic-table
    

<table class="nypl-basic-table">
  <thead>
    <tr>
      <th>One</th>
      <th>Two</th>
      <th>Three</th>
      <th>Four</th>
    </tr>
  </thead>
  <tr>
    <td>some value</td>
    <td>some value</td>
    <td>some value</td>
    <td>some value</td>
  </tr>

</table>

Search results summary

Presents a comma-separated list of search terms or filters along with the number of results. The result count has a special span tag (should be number-formatted e.g.: 1,232). Each applied facet or term has a button element to remove the facet and refresh the search results. the button contains the screen-reader-friendly text inside a span tag.

NOTE: Avoid adding a space between the facet and the remove button so that it won't wrap to the next line.

After the results summary, a “Start over” button should be added to allow the user to clear all facets and filters and start a new search from scratch.

Example
1,807 resources found for Young Adult , Fiction . 

.nypl-results-summary
.nypl-results-count
.nypl-facet
    

<div class="nypl-results-summary">
  <span class="nypl-results-count">1,807</span> resources found for
  <span class="nypl-facet">
    <strong>Young Adult<button><span>Remove Young Adult filter</span></button></strong>
  </span>,
  <span class="nypl-facet">
    <strong>Fiction<button><span>Remove Fiction filter</span></button></strong>
  </span>.
  <button>Start over</button>
</div>

Sorting

Sorters are nested inside a nypl-results-sorting-controls element. The sorters themselves implement a nypl-results-sorter microformat and use the aria-expanded attribute that must be modified accordingly by the appropriate JavaScript. The sorting options are nested in an unordered list with a menu role.

Example
  • relevance
  • title A-Z
  • title Z-A
  • date (old to new)
  • date (new to old)

.nypl-results-sorting-controls
.nypl-results-sorter
    

<div class="nypl-results-sorting-controls">
  <div class="nypl-results-sorter">
    <button aria-expanded="false">
    Sort by <strong>relevance</strong>
    <svg aria-hidden="true" class="nypl-icon" preserveAspectRatio="xMidYMid meet" viewBox="0 0 68 24">
      <polygon points="67.938 0 34 24 0 0 10 0 34.1 16.4 58.144 0 67.938 0" />
    </svg>
    </button>
    <ul class="hidden" role="menu">
      <li><a href="?sort=relevance"><strong>relevance</strong></a></li>
      <li><a href="#">title A-Z</a></li>
      <li><a href="#">title Z-A</a></li>
      <li><a href="#">date (old to new)</a></li>
      <li><a href="#">date (new to old)</a></li>
    </ul>
  </div>
</div>
